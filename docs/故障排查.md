# 故障排查

## 1. 启动后看不到窗口
先检查：

- 是否已经有旧进程在运行
- 是否启动到了纯时间模式
- 是否被最小化到了托盘

可查看：

- `startup.log`
- 托盘隐藏图标区域

如果窗口位置异常：

1. 打开设置窗口。
2. 点击“重置窗口位置”。
3. 保存后重开软件。

## 2. 纯时间模式倒计时结束没有提醒
当前版本已经修复了“主窗口未显示时提醒窗口设置 Owner 失败”的问题。

如果仍遇到异常：

1. 查看 `%LOCALAPPDATA%\MinimalSnapTimer\logs\startup.log`
2. 确认是否关闭了提醒弹窗
3. 确认系统通知是否被系统策略拦截

即使提醒弹窗失败，程序也应降级到通知、托盘气泡或日志记录，而不应再误报“软件启动失败”。

## 3. 下拉框点不开
当前版本已经修复 `ComboBox` 展开模板。

如果仍异常：

1. 先切换一次浅色 / 深色主题
2. 关闭软件后重开
3. 查看是否有旧版残留文件被误用

## 4. 纯时间模式左上角出现白色圆角块
当前版本已经修复这个问题。

根因是纯时间窗口左上角原先放了一个常驻状态徽标，在某些主题或前景色组合下只剩下背景胶囊可见。

修复后：

- 默认只显示时间
- 状态信息收纳到鼠标悬停时出现的工具条中

如果你仍看到异常块：

1. 确认当前 exe 已更新到最新发布包
2. 关闭软件后重新启动
3. 切换一次主题后再进入纯时间模式
4. 查看 `startup.log`

## 5. 托盘图标看不到
先展开 Windows 托盘隐藏图标区域。

如果仍看不到：

1. 查看 `startup.log`
2. 确认当前不是系统把图标折叠隐藏
3. 重新最小化到托盘一次

程序当前会优先使用 exe 图标作为托盘图标；如果加载失败，会自动回退到内置图标或系统默认图标。

## 6. 点击穿透后无法操作窗口
这是当前设计行为，不是缺陷。

- 开启点击穿透后，纯时间窗口本身不响应鼠标
- 必须通过托盘菜单关闭点击穿透

## 7. exe 为什么很大
当前只发布完整版 self-contained 单文件 exe，不提供 lite 包。

这是为了：

- 免安装运行时
- 降低用户使用门槛
- 保持发布环境稳定

## 8. 配置损坏怎么办
程序会自动：

1. 备份坏掉的 `settings.json`
2. 回退到默认配置
3. 尽量继续启动

## 9. 点击“设置”后提示只读属性绑定异常
当前版本已经修复设置窗口高级页只读展示字段误用默认双向绑定的问题。  
如果你仍看到类似 `ConfigDirectoryPath`、`LogDirectoryPath`、`ExecutablePath` 的只读属性绑定错误：

1. 确认当前 exe 已更新到最新发布包
2. 完全退出旧进程后重新启动
3. 查看 `startup.log`
