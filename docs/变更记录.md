# 变更记录

## v0.1.0-beta

### 本轮修复
- 修复从托盘菜单打开设置时，主窗口隐藏或未显示导致 `SettingsWindow.Owner` 设置失败的问题
- 设置窗口现在只有在存在可见且已加载的窗口时才设置 Owner，否则会无 Owner 居中显示
- 保留托盘恢复、纯时间模式、点击穿透和启动保护逻辑不变

### 本轮审计与轻量化优化
- 对计时器、托盘、主题和通知链路做了一次轻量化审计，未发现明显持续增长型内存泄漏
- 将托盘状态刷新从“每秒重建整套菜单”优化为“仅更新状态与剩余时间文本”，减少常驻分配和 UI 对象重建
- 普通日志增加大小滚动和数量清理，避免长期运行时日志文件无限累积
- 通知服务改为懒创建 `MediaPlayer`，避免未播放自定义声音时提前占用多媒体对象
- 实测发布版私有内存在常见场景下大多稳定在约 `57–58 MB`，对 `.NET 8 + WPF + 托盘 + 主题 + 多语言` 小工具来说属于合理范围
- 不建议为了当前这一级别的资源占用从 0 重写；若未来追求极致轻量化，可再评估原生技术路线

### 本轮优化
- 优化纯时间模式的布局留白，缩小时间内容与四周边框之间的空白
- 收紧纯时间窗口默认尺寸和悬停工具条边距，让纯时间模式整体更紧凑
- 保持时间文字大小计算逻辑不变，不通过缩小字体伪装“收紧”

### 本轮修复
- 修复设置窗口“高级”页只读展示字段仍使用 `TextBox.Text` 默认双向绑定，导致点击“设置”时报 `SettingsViewModel.ConfigDirectoryPath` 只读属性绑定异常的问题
- 将版本号、配置版本、日志目录、配置目录、当前 exe 路径等只读显示字段统一改为 `Mode=OneWay`
- 保留路径可复制能力，同时避免 WPF 对只读属性执行回写

### 本轮修复
- 修复纯时间模式左上角异常白色胶囊/空白徽标显示问题
- 将纯时间窗口状态徽标从默认常驻显示改为收纳进悬停工具条
- 保持点击穿透、多显示器恢复、提醒弹窗链路和托盘恢复逻辑不回退

### 本轮收口
- 修复纯时间模式下倒计时结束提醒不弹窗、后续触发 Owner 异常的问题

- 主窗口完成产品化重排
