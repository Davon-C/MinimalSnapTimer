# 发布说明 v0.1.0-beta

## 当前发布策略
- 仅提供完整版单文件 exe
- 不再提供 lite 包
- 默认面向普通用户直接分发

发布文件：

- `MinimalSnapTimer_v0.1.0-beta_win-x64.exe`

## 本轮重点改动
- 修复点击“设置”后高级页只读字段默认双向绑定导致的窗口打开异常
- 版本号、配置版本、日志目录、配置目录、当前 exe 路径现在均使用只读单向显示绑定
- 修复纯时间模式默认显示中左上角异常白色控件残片问题
- 纯时间窗口默认只保留时间显示，状态信息收纳到悬停工具条中
- 修复纯时间模式下提醒弹窗 Owner 异常
- 避免提醒链路 fire-and-forget 异常被误报为“软件启动失败”
- 主窗口 UI 重排，突出时间、状态和主操作
- 设置窗口按“通用 / 计时 / 外观 / 纯时间窗口 / 提醒 / 高级 / 预设”重新分组
- 统一深色主题常用控件样式
- 纯时间窗口补充悬停工具条和状态徽标
- 托盘菜单顶部显示当前状态和剩余时间
- 配置文件增加 `settingsVersion`

## 为什么只保留完整版
原因很直接：

- 普通用户无需安装 .NET Desktop Runtime
- 分发更简单
- 环境差异更少
- 手工验收路径更稳定

## 为什么 exe 体积较大
因为当前发布的是：

- `.NET 8`
- `WPF`
- `win-x64`
- `self-contained`
- `single-file`

运行时和 WPF 依赖会一起打包进 exe，所以体积较大属于正常现象。

## 当前已知限制
- 跟随系统主题不保证实时监听系统变化
- English 下少量边角文本仍可能未完全覆盖
- 原生 Toast 仍是轻量实现
